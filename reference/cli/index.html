Global Flags

These flags are available for all HTTP method commands:

Flag Short Type Default Description
--scope -s string (auto-detect) OAuth scope for authentication. Auto-detected from URL if not provided.
--no-auth bool false Skip authentication (no bearer token). Automatically set for HTTP URLs.
--header -H string[] [] Custom headers (repeatable, format: Key:Value).
--data -d string Request body (JSON string).
--data-file string Read request body from file (also accepts @{file} shorthand).
--output-file string Write response to file (raw for binary content).
--format -f string auto Output format: auto, json, raw.
--verbose -v bool false Verbose output (show headers, timing, redacted tokens).
--paginate bool false Follow continuation tokens/next links when supported.
--retry int 3 Retry attempts with exponential backoff for transient errors.
--binary bool false Stream request/response as binary without transformation.
--insecure -k bool false Skip TLS certificate verification (use only for testing).
--timeout -t duration 30s Request timeout (e.g., 30s, 5m).
--follow-redirects bool true Follow HTTP redirects automatically.
--max-redirects int 10 Maximum redirect hops before stopping.

Scope Detection

azd rest automatically detects the correct OAuth scope based on the URL. Supported services include:

  • Management API (management.azure.com)
  • Storage (*.blob.core.windows.net, *.queue.core.windows.net, etc.)
  • Key Vault (*.vault.azure.net)
  • Microsoft Graph (graph.microsoft.com)
  • Azure DevOps (dev.azure.com, *.visualstudio.com)
  • And 15+ more Azure services

For non-Azure endpoints, use --scope to provide the scope manually, or --no-auth to skip authentication entirely.
